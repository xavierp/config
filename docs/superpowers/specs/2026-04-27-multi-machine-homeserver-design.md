# Multi-Machine + Home Server Design Spec

## Goal

Extend the existing single-machine Nix flake to support a second host: a NixOS home server running a media stack, DNS ad-blocking, and home automation. The MacBook config stays untouched. Shared configuration is extracted into common modules.

## Architecture

### Flake structure

The existing flake gains a `nixosConfigurations."server"` alongside the current `darwinConfigurations."macbook"`. Both share common modules (locale, timezone, nix settings, user).

```
config/
├── flake.nix                      # darwinConfigurations.macbook + nixosConfigurations.server
├── hosts/
│   ├── macbook.nix                # existing nix-darwin config (unchanged)
│   ├── server.nix                 # NixOS system config (networking, firewall, storage)
│   └── server-hardware.nix        # auto-generated by nixos-generate-config
├── common/
│   └── default.nix                # shared: locale, timezone, nix settings, user "x"
├── home/
│   ├── default.nix                # macOS home-manager (unchanged)
│   └── server.nix                 # server home-manager: git, zsh basics, no GUI
├── services/
│   ├── arr.nix                    # Sonarr, Radarr, Prowlarr, Bazarr
│   ├── jellyfin.nix               # Jellyfin media server
│   ├── adguard.nix                # AdGuard Home DNS ad-blocker
│   └── homeassistant.nix          # Home Assistant
└── docs/
    └── superpowers/specs/...
```

### Common module (`common/default.nix`)

Extracted from both hosts to avoid duplication:
- Locale: `fr_FR.UTF-8`
- Timezone: `Europe/Paris`
- Nix settings: experimental features (flakes, nix-command), gc schedule
- User `x` with home directory and shell

The macbook host imports `common/` and adds its darwin-specific config. The server host imports `common/` and adds NixOS-specific config.

### Server host (`hosts/server.nix`)

NixOS configuration for the home server:
- **Networking**: static IP on local network, hostname, firewall with open ports per service
- **SSH**: enabled, key-only auth (password disabled), authorized key from MacBook
- **Storage**: mount point for media drives (assumed `/mnt/media`), directory structure for downloads/library
- **Services**: imports from `services/` directory

### Hardware config (`hosts/server-hardware.nix`)

Auto-generated by `nixos-generate-config` on the actual server hardware. Contains:
- Filesystem mounts
- Boot loader config (GRUB or systemd-boot depending on hardware)
- Kernel modules for specific hardware

This file is generated on the server, then copied into the repo.

## Services

### *arr stack (`services/arr.nix`)

All services run as NixOS systemd services via nixpkgs:
- **Sonarr**: TV show management → port 8989
- **Radarr**: Movie management → port 7878
- **Prowlarr**: Indexer management → port 9696
- **Bazarr**: Subtitle management → port 6767

Shared group `media` for filesystem permissions. All data under `/mnt/media/`:
```
/mnt/media/
├── downloads/       # download client output
├── movies/          # Radarr library
├── tv/              # Sonarr library
└── music/           # future use
```

A download client (Transmission or qBittorrent) is needed. **Transmission** is in nixpkgs and simpler to configure declaratively.

### Jellyfin (`services/jellyfin.nix`)

- Port 8096 (HTTP)
- Media libraries point to `/mnt/media/movies` and `/mnt/media/tv`
- Runs as `jellyfin` user, member of `media` group
- Hardware transcoding: depends on server GPU — if Intel, enable VA-API; if none, software only

### AdGuard Home (`services/adguard.nix`)

- Port 3000 (web UI), port 53 (DNS)
- Upstream DNS: Cloudflare (1.1.1.1) + Google (8.8.8.8)
- Default blocklists: AdGuard default + OISD
- Router DHCP should point clients to server IP as DNS

### Home Assistant (`services/homeassistant.nix`)

- Port 8123
- NixOS has `services.home-assistant` with declarative config
- Initial setup is mostly through the web UI — Nix manages the service lifecycle, not the automations
- Consider: run in a container instead if HA's nixpkgs module is too restrictive for integrations

## Server home-manager (`home/server.nix`)

Minimal profile for SSH sessions:
- `programs.zsh` with basic config (no starship, no fzf, no tmux — keep it light)
- `programs.git` with same identity as MacBook
- Shell aliases subset (g, dk, dkc)

## Deployment

### Phase A (initial): Local rebuild

1. Install NixOS on the server (USB installer)
2. Clone the config repo on the server
3. `sudo nixos-rebuild switch --flake ~/src/config#server`

### Phase B (future): Remote deploy from MacBook

Add `deploy-rs` to the flake:
- `deploy.nodes.server` with SSH connection details
- Run `deploy .#server` from the MacBook
- Requires the server's SSH host key in known_hosts

Not in scope for the initial implementation — just ensure the flake structure doesn't block it.

## Firewall

Server firewall allows only:
- SSH (22)
- DNS (53) — for AdGuard
- HTTP services on LAN only (8096, 8989, 7878, 9696, 6767, 3000, 8123)
- No public-facing ports unless behind a reverse proxy (future: Caddy or nginx)

## What's NOT in scope

- VPN / remote access (Tailscale or WireGuard — future addition)
- Reverse proxy with HTTPS (Caddy — future addition)
- Monitoring / alerting (Grafana/Prometheus — future addition)
- Backup strategy for server data
- Download client VPN (for privacy on torrents)
- The actual NixOS installation (documented as a prerequisite, not automated)
